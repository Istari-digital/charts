# FIPS 140-3 Compliance Guide: charts

## Overview

This repository contains Helm charts (istari-platform, istari-zitadel-configurator) that deploy Kubernetes services. The charts themselves do not perform cryptographic operations -- compliance depends on the container images and infrastructure they deploy on.

## Container Images Requiring FIPS Variants

| Component | Image | Required FIPS Action |
|-----------|-------|---------------------|
| fileservice2 | istaridigital.jfrog.io/.../fileservice2 | Use FIPS base image with OpenSSL 3.x FIPS provider |
| mcp-service | istaridigital.jfrog.io/.../mcp-service | Same as fileservice |
| frontend-service | istaridigital.jfrog.io/.../frontend-service | Low risk (static web serving, crypto in browser) |
| docs-service | istaridigital.jfrog.io/.../docs-service | Low risk (static content) |
| zitadel-configurator | Terraform + kubectl container | Build Terraform with Go BoringCrypto |
| Zitadel (upstream) | Configured by zitadel-configurator | Must be built with GOEXPERIMENT=boringcrypto |

## TLS Configuration Requirements

- All service-to-Zitadel communication must use TLS (the `zitadel_insecure` value must remain `false`)
- SSL_CERT_FILE, SSL_CERT_DIR, and GRPC_DEFAULT_SSL_ROOTS_FILE_PATH are set for trusted certificate injection
- SMTP connections must use TLS (`smtp.tls: true`)

## OIDC/JWT Configuration

- JWT access tokens (RS256/ES256) are FIPS-approved algorithms
- Private Key JWT authentication uses RSA keys generated by Zitadel
- JWKS validation endpoints must be accessed over HTTPS

## Infrastructure Prerequisites

- Kubernetes etcd encryption-at-rest with FIPS-validated KMS provider
- Istio service mesh with FIPS-validated Envoy proxies for mTLS
- Database connections must use TLS with `sslmode=require` minimum

## Rules for Contributors

1. Do not set `zitadel_insecure: true` in production configurations
2. All new container image references must have a FIPS variant documented
3. Do not add services that connect to databases without enforcing TLS
4. OIDC application `dev_mode` must be `false` in production
5. All redirect URIs must use HTTPS scheme
6. Changes to TLS, OIDC, or JWT configuration require updating this document
